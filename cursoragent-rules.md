# CURSOR AGENT RULES

## GENERAL PRINCIPLES

Read all project files before writing code.

Never ignore:

* content.md

The hero image is the master visual reference.

---

# NO HALLUCINATIONS

Never invent:

* text
* images
* sections
* categories
* prices
* contacts
* schedules
* file names
* colors

If information is missing, ask or leave TODO.

Never generate placeholder content unless explicitly requested.

Bad:

"Lorem ipsum"

Bad:

"teacher.jpg"

Good:

TODO: teacher photo missing.

---

# IMPLEMENTATION STRATEGY

Never build the entire website at once.

Work step by step:

1. Analyze.
2. Plan.
3. Implement.
4. Review.
5. Continue.

Stop after each major section.

---

# DESIGN CONSISTENCY

All pages must inherit:

* colors
* shadows
* typography
* atmosphere
* animations

The Hero section is the visual master.

Never introduce:

* white backgrounds
* blue gradients
* corporate UI
* bootstrap style
* new button styles

---

# FILE SAFETY

Never delete:

* content.md
* images
* public files

Never overwrite existing content without permission.

---

# CODE QUALITY

Use:

* TypeScript
* strict typing
* reusable components
* semantic HTML

Avoid:

* duplicated code
* inline styles
* hardcoded values

---

# CONTENT RULES

Text comes only from:

* content.md


Never rewrite marketing text.

Never shorten content.

Never invent translations.

---

# RESPONSIVENESS

Desktop first.

Mobil second.

Tablet third.

Every section must be responsive.

---

# ANIMATIONS

Use:

* Framer Motion

Avoid:

* bouncing
* shaking
* excessive movement

Animation should feel:

* elegant
* cinematic
* slow

---

# BEFORE EDITING

Before modifying existing code:

1. Read the file.
2. Explain the change.
3. Apply the change.

---

# BEFORE CREATING NEW FILES

Ask:

Can the existing component be reused?

Prefer refactoring over duplication.

---

# IF CONFLICT EXISTS

Priority:

1. content.md
2. existing code
3. agent assumptions

---

# ABSOLUTE RULE

Do not assume.

Do not invent.

Do not simplify.

Do not redesign without request.
